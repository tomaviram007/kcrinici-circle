
# הצגת תפריט לכולם עם פופאפ לגולשים שלא מחוברים

## מה ישתנה

### Header.tsx - הצגת כל הקישורים תמיד
- התפריט (ראשי, מודעות, דרושים, חברים, אירועים) יוצג תמיד - גם למי שלא מחובר
- קישורי Admin ("שולחן המנהל") ימשיכו להופיע רק לאדמינים
- כשגולש לא מחובר (או לא מאושר) לוחץ על קישור מוגן, במקום לנווט - יופיע דיאלוג (פופאפ) עם הודעה: "תוכן זה זמין לחברי המועדון בלבד. הצטרף למועדון כדי לגשת לכל התכנים"
- הפופאפ יכלול כפתור "הצטרפות" שמפנה לדף הרישום, וכפתור "כניסה" למי שכבר רשום
- כפתורי "כניסה" ו"הצטרפות" בצד שמאל ימשיכו להופיע למי שלא מחובר

## פרטים טכניים

### קובץ: `src/components/layout/Header.tsx`

**שינויים:**
1. ייבוא של `Dialog` מ-`@/components/ui/dialog`
2. הוספת state: `showMemberDialog` (boolean)
3. רשימת `navLinks` תציג תמיד את כל הקישורים (ראשי, מודעות, דרושים, חברים, אירועים) - ללא תלות ב-`isApproved`
4. קישורי Admin ימשיכו להיות מותנים ב-`isAdmin`
5. בלחיצה על קישור מוגן: אם המשתמש לא מחובר או לא מאושר - `preventDefault` + פתיחת דיאלוג
6. אם המשתמש מחובר ומאושר - ניווט רגיל

**לוגיקת הקליק:**
```text
לחיצה על קישור מוגן (dashboard/announcements/jobs/members/events):
  - אם user מחובר ומאושר -> ניווט רגיל
  - אם user לא מחובר או לא מאושר -> מניעת ניווט + פתיחת פופאפ
```

**תוכן הפופאפ:**
- כותרת: "תוכן לחברי מועדון"
- טקסט: "התוכן הזה זמין לחברי המועדון בלבד. הצטרף אלינו כדי לגשת לכל התכנים הבלעדיים."
- כפתור ראשי: "הצטרפות למועדון" (מפנה ל-/register)
- כפתור משני: "כבר רשום? כניסה" (מפנה ל-/login)

### תצוגה בדסקטופ ובמובייל
- התפריט יוצג תמיד (הסרת התנאי `user &&` מהתצוגה)
- בניידים - תפריט המבורגר יעבוד גם בלי חיבור
- הפופאפ יופיע גם בדסקטופ וגם בנייד
