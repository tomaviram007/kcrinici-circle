
# הוספת סקשן "הפרסומים שלי" בעמוד הפרופיל

## מה ייבנה
סקשן חדש בתחתית עמוד הפרופיל (מתחת לשינוי סיסמה) שיציג למשתמש את כל מה שהוא פרסם באתר, עם אפשרות לנהל את הפרסומים.

## מבנה הסקשן

הסקשן יכלול **טאבים** לפי סוג תוכן:
- **מודעות** - כל המודעות/מכירות שפרסם (כולל ממתינות לאישור)
- **משרות** - כל המשרות שפרסם
- **אלבומים** - כל אלבומי הגלריה שיצר

כל טאב יציג רשימה עם:
- כותרת הפרסום + תאריך
- סטטוס (ממתין לאישור / מאושר / נמכר)
- כפתור "סמן כנמכר" (למודעות מכירה שטרם נמכרו)

## שינויים נדרשים ב-RLS

צריך לאפשר למשתמשים לקרוא את המודעות והמשרות **שלהם** גם אם עדיין לא אושרו. כרגע ה-SELECT policies מאפשרות לקרוא רק פרסומים מאושרים. נוסיף policies חדשות:
- `announcements`: משתמש יכול לקרוא מודעות שהוא יצר (`created_by = auth.uid()`)
- `jobs`: משתמש יכול לקרוא משרות שהוא יצר (`created_by = auth.uid()`)

## פרטים טכניים

### 1. מיגרציה - RLS policies חדשות
```sql
CREATE POLICY "Users can read own announcements"
  ON public.announcements FOR SELECT
  USING (auth.uid() = created_by);

CREATE POLICY "Users can read own jobs"
  ON public.jobs FOR SELECT
  USING (auth.uid() = created_by);
```

### 2. עדכון `src/pages/Profile.tsx`
- ייבוא רכיבי Tabs מ-Radix UI
- הוספת state לנתוני הפרסומים (מודעות, משרות, אלבומים)
- שליפת הנתונים לפי `created_by = userId` בעת טעינת הפרופיל
- הצגת טאבים עם רשימת פרסומים לכל סוג
- כפתור "סמן כנמכר" למודעות מכירה עם עדכון ב-DB
- תגיות סטטוס צבעוניות (ממתין / מאושר / נמכר)
- הודעה "אין פרסומים עדיין" כשאין תוכן
